# TRABALHO FINAL


## Desenvolvimento Rapido de Aplicacoes em Python (RAD)

- **Disciplina:** Desenvolvimento Rapido de Aplicacoes em Python
- **Professor:** Elton Silva
- **Semestre:** 2026_01
- **Valor total:** 5,0 pontos

## 1. Objetivo do Trabalho

Desenvolver uma aplicacao utilizando a metodologia RAD (Rapid Application Development), implementando um sistema funcional que contemple:

- Interface grafica (Web ou Desktop)
- Backend em Python utilizando framework
- Banco de dados relacional
- CRUD completo (Create, Read, Update, Delete)

O projeto devera demonstrar claramente a aplicacao das fases do RAD e a integracao entre interface, logica de negocio e persistencia de dados.

## 2. Requisitos Obrigatorios

O sistema desenvolvido devera obrigatoriamente:

- Utilizar um framework Python (exemplos: Flask ou Django).
- Utilizar banco de dados relacional (SQLite, PostgreSQL ou MySQL).
- Implementar CRUD completo.
- Possuir no minimo 2 tabelas relacionadas.
- Realizar validacao de dados (ex: campos obrigatorios, formatos validos).
- Entregar relatorio conforme modelo fornecido.

`Deploy` nao e obrigatorio.

## 3. Aplicacao da Metodologia RAD

O projeto devera evidenciar claramente as quatro fases do RAD:

### Fase 1 - Planejamento de Requisitos

- Definicao do problema
- Objetivo do sistema
- Requisitos funcionais
- Requisitos nao funcionais
- Tecnologias escolhidas
- Modelo inicial do banco de dados

### Fase 2 - Design do Usuario

- Prototipos de tela (wireframe ou esboco)
- Estrutura de navegacao
- Decisoes de interface
- Ajustes realizados apos discussao em equipe

### Fase 3 - Construcao

- Implementacao do backend
- Implementacao do banco
- Implementacao da interface
- Integracao entre as camadas
- Registro de desafios tecnicos enfrentados

### Fase 4 - Transicao

- Testes realizados
- Correcoes implementadas
- Validacao do funcionamento do sistema

## 4. Cronograma de Entregas

| Etapa | Entrega | Pontos |
|---|---|---|
| Fase 1 - Planejamento | Documento de requisitos + modelo inicial do banco | 1,0 |
| Fase 2 - Design | Prototipos e decisoes de interface | 1,0 |
| Fase 3 - Construcao | Sistema com CRUD funcional e banco integrado | 2,0 |
| Relatorio Final + Apresentacao | Documento completo + demonstracao do sistema | 1,0 |
| **Total** |  | **5,0** |

As entregas serao realizadas conforme datas informadas em sala.

## 5. Criterios de Avaliacao

### Planejamento (1,0 ponto)

- Clareza dos requisitos
- Coerencia do modelo de dados
- Adequacao das tecnologias escolhidas

### Design (1,0 ponto)

- Organizacao da interface
- Coerencia com os requisitos
- Justificativa das decisoes tomadas

### Construcao (2,0 pontos)

- CRUD completo funcionando
- Integracao com banco de dados
- Validacao e tratamento de erros
- Organizacao e estrutura do codigo

### Relatorio e Apresentacao (1,0 ponto)

- Aplicacao clara das fases do RAD
- Documentacao organizada
- Clareza na apresentacao do sistema

## 6. Relatorio Final

O relatorio devera seguir o modelo disponibilizado pelo professor, contendo obrigatoriamente:

- Informacoes gerais do projeto
- Introducao
- Planejamento de requisitos
- Design do usuario
- Construcao
- Transicao
- Conclusoes
- Link para o repositorio (se aplicavel)

### Formatacao

- Fonte Arial ou Calibri
- Tamanho 12
- Espacamento 1,5
- Texto justificado
- Padrao de citacao: ABNT ou APA

## 7. Apresentacao Final

Cada grupo devera:

- Demonstrar o funcionamento do sistema
- Explicar como aplicou as fases do RAD
- Apresentar estrutura do banco
- Mostrar organizacao do codigo

Todos os integrantes deverao participar da apresentacao.

## 8. Orientacoes Importantes

- O projeto devera ser desenvolvido em grupo.
- A divisao de tarefas devera estar registrada no relatorio.
- Sistemas incompletos ou sem integracao com banco terao reducao significativa de nota.
- Trabalhos entregues fora do prazo poderao sofrer penalidade.

## 9. Consideracoes Finais

Este trabalho tem como objetivo integrar teoria e pratica, simulando um cenario real de desenvolvimento de software utilizando ciclos iterativos e incrementais, conforme a metodologia RAD.

O foco nao e apenas "fazer funcionar", mas demonstrar planejamento, organizacao e aplicacao estruturada do processo de desenvolvimento.
